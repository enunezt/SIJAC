El Sistema de Información de las Juntas de Acción Comunal “SIJAC”, es un Producto Software desarrollado con el fin de dar solución tecnológica al desarrollo de las actividades de estos organismos cívicos-sociales SIJAC se encuentra estructurado bajo la siguiente Arquitectura de información:

Módulo de Seguridad, este presenta las siguientes características:

a. Protección para la contraseña:

  • Cada uno de los usuarios finales debe escoger una palabra combinada con números, como palabra clave, luego debe memorizarla, posteriormente digitarla en el campo clave para ser admitido por el sistema
  • Se recomienda a los usuarios no escribir la contraseña, para evitar el riesgo que sea leída por un intruso
  • No se dejó despliegue para las claves
  • Se prohíbe la comunicación de las claves, solo en caso extremadamente necesario
  • Se recomienda a los usuario modificar la contraseña, luego que ha sido comunicada
  • Se recomienda el cambio de clave en un lapso de tiempo
  • Que la clave quede compuesta mínimo por ocho caracteres
  • Se dejó tres veces como intentos máximos fallidos de acierto, para ingresar la contraseña
b. Auditoría y controles de acceso
  • Permite el registro cada vez que ocurra un evento
  • Los registros se almacenan en un área protegida por el sistema, a donde tan solo tendrá acceso el administrador del sistema
  • Las revisiones se hacen en periodos no mayores de 30 días

c. Núcleos de seguridad, permite crear

  • Cuenta de usuario
  • Administrador de sistema
  • Perfiles de usuario
  • Usuario en el script de la BD SIJAC

Módulo de Negocio: En este modelo se sistematizaron los siguientes procesos:

  • a. Registro de datos de la JAC. Usuario: Administrador del sistema
  • b. Registro de datos de dignatarios: Usuario: Administrador del sistema
  • c. Registro de datos en el libro de afiliados. Usuario: El secretario (a)
  • d. Registro de datos de comités. Usuario: El secretario (a), presidente
  • e. Registro de pago de sostenimiento. Usuario: Tesorero
  • f. Registro de desafiliación. Usuario: El secretario (a) y comité
  • g. Genera la lista de afiliados activos para el control de asistencia en los diferentes tipos de reuniones.
    • El sistema genera un formato de planilla para ser diligenciado por los afiliados activos presentes en la reunión
    • Genera planilla de afiliados no asistentes a la reunión. Usuario: secretario(a)
  • h. Registra el acta de reunión de Asamblea Ordinaria. Usuarios: secretario(a) y el presidente
  • i. Registra el Acta de reunión de asamblea General extraordinaria. Usuarios: secretario(a) y el presidente
  • j. Registra el informe general de gestión. Usuarios: secretario(a) y el presidente
  • k. Registra el Acta de Proceso electoral, para hacer la elección o conformación del Tribunal de Garantía. Usuarios: secretario(a), el presidente y El Tribunal de Garantía.
  • l. Registra las planchas para elecciones. Usuario: Tribunal de garantía y CC.
  • m. Registra Acta de nombramiento de dignatarios. Usuarios: secretario(a),l presidente y El Tribunal de Garantía.
  • n. Registra el Acta de Empalme de Dignatarios. Usuarios: Dignatarios salientes y Dignatarios entrantes.
  • o. Genera certificados para el afiliado. Usuarios: secretario(a), presidente y afiliado.
  • El sistema genera los reportes necesarios para rendir la información pertinente sobre el desarrollo de las funciones de la JAC.

Bases de Datos El sistema de gestión de la DB de SIJAC es en Postgres, este sistema es estable, de alto rendimiento, flexible, y permite extenderse fácilmente.
